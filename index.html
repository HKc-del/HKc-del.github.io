字符串函数匹配和替换字符串

在字符串中查找字符串的函数strstr()、strchr()、strrchr()、stristr()

strstr()函数的原型如：

string strstr(string haystack,string needle[,bool before_needle=false])

haystack 表示要被搜索的字符串，before_needle表示返回那部分的字符串，为true时返回needle之前的部分字符串，为false时返回needle之后的字符串，返回第一个匹配的位置。

stristr()将不分大小。strrchr()将返回最后一个匹配的needle的位置。

正则表达式

正则表达式是一种描述文本所包含模式的方法，描述了一种文本的匹配模式，在PHP中，匹配正则表达式的模式更像是strstr()的匹配模式，而非==的匹配模式。

分隔符

每个正则表达式必须被包含在一对分隔符之中，可以选择非字母、数字、\、和空格的任意符号作为分隔符。一般使用的分割符为/。分隔符后可以添加模式修饰符。

子表达式

()括号中，可以被视为一个表达式与计算中的()相同。例：/(very )* large/可用于匹配"large"、"very large"、"very very large"等

子表达式计数

可以用{}指定子表达式可以出现的次数。例：/(very ){1,3}/表示要出现1-3次，若没有指定上限，则表示至少出现几次，也可以只用一个数字指定出现的次数。

定位到字符串的开头与结尾

^表示该模式的匹配必须出现在字符串的起始位置，$表示该模式出现在字符串的结尾。

在匹配特殊字符时，必须在字符前加\作为转义。在PHP中，必须使用单引号来进行模式的引用，如果使用双引号则必须使用双反斜杠进行转义，即使用\\ \\php解释器像将其翻译为\\正则解释器再将其翻译为\。

该函数具有一个必选参数以及三个可选参数。

第一个必选参数即string，为目标字符串。第二个参数为可选参数flags，指定了该函数将如何进行翻译，可以使用位掩码来表示可能的组合值。其中flags的可选值如下表所示：

标志 意义

ENT_NOQUOTES 不对单引号和双引号编码
ENT_QUOTES 对双引号和单引号编码
ENT_HTML401 将代码视作HTML 4.01
ENT_XML1 将代码当作XML 1
ENT_XHTML 将代码视作XHTML
ENT_HTML5 将代码视作HTML 5
ENT_IGNORE 不返回空字符串，并且忽略无效的代码序列
ENT_SUBTITUTE 用unicode替换字符替换无效的代码单元序列
ENT_DISALLOWED 用unicode替换无效的代码点

第三个为可选参数encoding即编码，默认为UTF-8

第四个为可选参数，制定了为是否需要对HTML实体进行编码，默认值为真。

1. <script>标签的使用

<script>是向HTML标签中添加脚本的主要方法，这个元素最早由Netscape创造。

​ HTML 4.01 规定了如下6个元素。

​ async：可选，表示立即下载该脚本，但是不阻塞页面中的其他操作，例如下载其他资源、等待加载其他脚本，只对外部脚本起作用。

​ charset：可选，表示通过src获得的脚本文件的字符集，但大部分浏览器会忽略其值，该属性很少人使用

​ defer：可选。表示脚本可以延迟到文档被完全解析后和显示后，在进行执行，只对外部脚本有效

​ language：已废弃，表示用于编写代码的语言。现已被大部分浏览器忽略

​ src：可选。表示包含着要执行代码的外部文件。

​ type：可选。表示编写代码的内容类型(也称MIME类型)，text/javascript已经不被人推荐使用，服务器在传送使用的MIME类型通常是application/x-javascript，但是在脚本中设置该值可能导致脚本被忽略。考虑到约定俗成和最大限度的浏览器兼容性，目前该值仍旧填写text/javascript，不过这个值不是必须的，没有指定该属性，择其默认值为text/javascript

进程管理中的数据结构

进程控制块

作用：

  1. 独立运行基本单位的标志
  2. 能实现间断性运行方式
  3. 提供进程管理所需要的信息
  4. 提供进程调度所需要的信息
  5. 实现与其它进程的同步与通信

内容：

  1. 进程标识符信息

    进程标识符用于惟一地标识一个进程。一个进程通常有两种标识符

    1. 内部标识符 为每一个进程赋予一个惟一的数字标识符
    2. 外部标识符 它由创建者提供(进程的名字)，通常是由字母、数字组成，往往是由用户（进程）在访问该进程时使用

  2. 处理器状态信息

    处理机状态信息主要是由处理机的各种寄存器中的内容组成的

  3. 进程调度信息

    1. 进程状态。指明进程的当前状态
    2. 进程优先级
    3. 进程调度所需的其它信息 进程已等待CPU的时间总和、进程已执行的时间总和等
    4. 事件 即 阻塞原因 通过PCB中存储的该进程的阻塞事件，找到该进程所在的阻塞队列

  4. 进程控制信息

    1. 程序和数据的地址
    2. 进程同步和通信机制
    3. 资源清单
    4. 链接指针

实时操作系统中没有进程的概念，调度单位是任务，相当于线程

作业调度的过程

将程序装入内存->创建进程->进入就绪队列->被调度->执行

C语言中的多线程

在linux下进行编程需要使用头文件pthread.h，在编译时同时要加上-lpthread选项，使gcc在编译时连接到这些库的二进制代码。

创建线程 pthread_create函数

pthread_create(&thrd1,NULL,(void*)&thread_fuction,(void*)&some_argument)

线程创建函数主要需要4个参数，第一个为线程变量名，即被创建的线程变量的地址，第二个参数为线成的属性指针，缺省值为NULL，第三个参数被创建的程序代码，即该线程所要执行的函数的函数指针，第四个参数为上述函数在执行时所需要的参数。

结束线程 pthread_exit函数

pthread_exit(void *retval);

retval变量用于存放线程结束的退出状态。

线程等待函数pthread_join函数

pthread_join (thrd1,NULL);

在运用pthread_create成功创建多个线程后用户并不知道哪个线程会先被执行。我们需要运用pthread_join函数来等待某个线程的结束。

课堂笔记（操作系统）（一）

进程

调度器

常驻内存，不是一个进程，但是可以被执行。

就绪状态的任务才能有资格被执行。

进程的特征

  1. 动态性 由”创建而产生，由调度而执行，有撤消而消亡“
  2. 并发性 _多个进程实体同存于内存之中，且能在一段时间内同时运行_。
  3. 独立性 进程实体是一个能独立运行
  4. 异步性 指进程按各自独立的、不可预知的速度向前推进，或说进程实体按异步方式运行

进程的三种状态

  1. 就绪状态 万事俱备，只差CPU

  2. 执行状态 正在执行

  3. 阻塞状态 _当前正在执行的进程由于某事件而暂时无法继续执行时，便进入暂停状态，即阻塞状态_。

    往往根据阻塞原因，系统中会设置多个阻塞队列，当不同的阻塞原因解除时，方便调度不同的进程执行。

    时间片到达而被抢占CPU任务从执行状态变为就绪状态

    sleep休眠，那么将从运行态变为阻塞态(等待态)

    等待态与阻塞态为一种状态

    如果执行状态的进程被中断，那么将会到就绪态

    进程状态只能同时处于一种状态

进程的创建与终值

  1. 创建状态 资源无法得到满足

  2. 终止状态

  3. 挂起状态 对任何状态的进程都可以挂起

    引入挂起状态的原因：

    1. 终端用户的请求
    2. 父进程的请求
    3. 负荷调节的需要，把一些不重要的进程挂起
    4. 操作系统的需求

桌面级DBMS与企业级DBMS系统比较

  1. 桌面级DBMS功能简单、软件体量小、处理能力弱、通常只支持单一语言的应用程序开发。例：Access
  2. 企业级DBMS功能完善、软件体量大、处理能力强、支持多种不同语言的应用开发.例：Oracle DB

关系数据库DBMS的特点

数据结构简单、操作灵活、支持关系集合运算

对象关系DBMS的特点

除具备关系数据库的特点以外还有面向对象的特点，例如继承、封装、以及对于复杂数据结构的管理

DBMS需要有多个版本以保证各不同平台上的兼容性。

Mysql不是企业级数据库，属于商业级数据库，处理能力、安全性、可靠性无法达到企业级的水平。

PostgreSQL数据库主要对象没有存储过程。

PostgreSQL采用客户服务器架构。

关系模型

关系特征

  1. 表中每行存储实体的一个示例数据
  2. 表中每列包含实体的一项属性数据
  3. 表中单元格只能存储单个值
  4. 不允许有重复的行
  5. 不允许有重复的列
  6. 列顺序可任意
  7. 行顺序可任意

​ 代理键 增减一个列，采用DBMS自动生成的序号作为主键。

数据结构、数据操作、数据关系约束(列取值类型，取值范围，取值是否唯一，是否允许空值，关联表参照完整性约束)

数据操作

集合运算操作：选择、投影、连接、交、并、差

针对元组的操作：插入、修改、删除

关系模型的完整性

完整性约束的作用
  1. 消除关系表的元组重复储存
  2. 保持关联表的数据一致性
  3. 实现业务数据规则
组成
1. 实体完整性约束
2. 参照完整性约束
3. 用户自定义完整性约束
实体完整性约束

指在关系表中实体的主键取值约束以保证关系表中的每个元组可被唯一标识

规则

  1. 主键属性列都不允许为空
  2. 主键取值唯一，并区分关系表中的每个元组
参照完整性约束

指在关系表中需要遵守的数据约束，以保证关系之间关联数据一致性

规则

​ 若关系R中的外键F与关系S中的主键相关联，则R中外键F值必须与S中主键K值一致。

外键

​ 在关联的两个关系中，他们具有一个或多个相同属性。若关联列在第一个关系中为主键，则在第二个关系中为外键

用户自定义完整性约束

指用户根据具体业务对数据处理规则要求所定义的数据约束

用户可定义的完整性约束

1. 定义列的数据类型和范围
2. 定义列的缺省值
3. 是否允许空值
4. 定义列取值唯一性
5. 定义列之间的数据依赖性

分组交换网络中的延迟

  1. dproc = 节点处理时延－nodal processing delay

    典型几个微秒或更小

  2. dqueue = 排队时延－queuing delay

    取决于路由器的拥塞程度

  3. dtrans = 传输时延－transmission delay

    = L/R, 低带宽链路比较显著

  4. dprop = 传播时延－propagation delay

    几个微妙到数百毫秒（卫星通信高传播延迟）

吞吐量

  1. 瞬时吞吐量(某一瞬间)

  2. 平均吞吐量(一段时间内)
