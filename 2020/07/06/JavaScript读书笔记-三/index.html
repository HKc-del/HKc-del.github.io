RegExp的实例属性

RegExp每个实例都有以下属性：

  • global 布尔值，表示是否设置了g标志
  • ignoreCase boolean 表示是否设置了i标识
  • lastIndex 表示开始搜索下一个匹配项的字符位置
  • multiline 是否设置了m标志
  • source 正则表达式的字符串表示

RegExp的实例方法

主要方法为exec()，该方法是为了捕获组(即：子匹配)而设计的，exec()只接受一个参数，即要应用模式的字符串，然后返回一个匹配项的数组，如果没有匹配将返回null，若匹配成功将返回，一个Array对象和两个额外的属性，即index与input分别表示第一处匹配的下标处和输入的字符串。而Array对象中将存放着捕获组的字符串。

全局模式与非全局模式的区别

若使用全局模式在执行exce()方法时，将不断地移动lastIndex，以向字符串后部移动，若不启用全局模式，将不会移动字符串的指针，始终只返回第一处匹配的模式。

第二个方法为test()方法，他将接受一个字符串参数，若存在匹配将返回true，若不存在匹配，则返回false，这个语句常被用于if语句之中。

RegExp的构造函数属性

构造函数属性在其他语言中被视为静态属性，他将返回作用域所有RegExp实例的一些特性。主要有以下属性：

  • input:短属性$_标识最近一次要匹配的字符串
  • lastMatch:短属性$&最近一次的匹配项
  • lastParen:短属性$+最近一次的匹配组
  • leftContext:短属性 $` 返回input中lastMatch之前的文本
  • multilin:短属性$*表示是否所有表达式都使用了多行模式
  • rightContext:短属性$'返回input中lastMatch后的文本
